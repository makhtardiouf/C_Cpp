Problem description

Telephone keypads are commonly used to input alphabetic characters into an electronic device, typically a phone. They have 8 buttons and each button has three to four alphabetic characters assigned as follows:

Button 1: ABC
Button 2: DEF
Button 3: GHI
Button 4: JKL
Button 5: MNO
Button 6: PQRS
Button 7: TUV
Button 8: WXYZ

When you push a button N times very quickly, the N-th character associated with the button is input. For example, to input GOOGLE, you should push button 3 (G) once, button 5 three times (O), button 5 three times again (O), button 3 once (G), button 4 three times (L), and button 2 twice (E).
Note: you can input consecutive characters that share a single button (e.g. S and P) by pausing for a short time between button pushes.

You are given one word. Your task is to calculate how many button pushes are required to input the given word.

Input

Test cases will be provided in the following multiline text format, using only ASCII characters. The first line contains one integer, C, which is the number of test cases that will follow. From the second line onwards, one test case is provided per line and each test case has one word.

Limits
All words in the input contain only capital letters (A-Z).
Number of test cases: 1 <= C <= 50.
Number of characters in a word: 1 <= Length(word) <= 100.
A word can be any sequence of capital letters, and need not be in English.
Output

For each test case, output the result in the following format:

Case #k: N

where k is the index of the test case, starting from 1, and N is the number of button pushes required to input the word.

Sample input (task3_small_sample.input.txt)
9
P
S
SP
GOOGLE
ALPHABET
AAAAA
ADGJMPTW
CFILOSVZ
ABCDEFGHIJKLMNOPQRSTUVWXYZ

Output for sample input (task3_small_sample.output.txt)
Case #1: 1
Case #2: 4
Case #3: 5
Case #4: 13
Case #5: 13
Case #6: 5
Case #7: 8
Case #8: 26
Case #9: 56
