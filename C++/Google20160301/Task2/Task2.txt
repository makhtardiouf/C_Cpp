Problem description

You are given two integers K and S, and a grid of integers with N rows and N columns. A grid is good if and only if it contains K vertical consecutive cells or K horizontal consecutive cells, whose sum is exactly S. Your task is to determine whether a given grid is good or not.

Input

Test cases will be provided in the following multiline text format, using only ASCII characters. The first line contains one integer, C, which is the number of test cases that will follow. The second line is blank. From the third line onwards, the test cases, separated by one blank line, will follow. Each test case has the following format.

N
g[1,1]  g[1,2] …  g[1,N]
g[2,1]  g[2,2] …  g[2,N]
…          …             ...
g[N,1] g[N,2] … g[N,N]
K S

N is the size of the grid. N lines representing an N-by-N grid will follow, and g[i,j] is the integer in cell (i, j).

All tokens in a line are separated by a single space.

Limits
All numbers in the input are integers.
Number of test cases: 1 <= C <= 40.
Size of grid: 2 <= N <= 15.
Integers in grid: 0 <= g[i,j] <= 9.
Number of cells to sum: 1 <= K <= N.
Target sum: 0 <= S <= 200.
Output

For each test case, output the result in the following format:

Case #k: ANSWER

where k is the index of the test case, starting from 1. ANSWER is “1” if the grid is good, and “0” otherwise.
